# Subtração — Complemento de 2 e Restore

<div className="section-chip">⚙️ Aritmética · SEL = 001</div>

## Teoria: Complemento de 2

A maneira mais elegante de implementar subtração em hardware é reutilizar o somador já existente. A técnica é o **complemento de 2**:

```
A − B  ≡  A + (complemento de 2 de B)
       ≡  A + (~B + 1)
```

Onde `~B` significa inverter todos os bits de B (complemento de 1), e depois somar 1. Isso é feito injetando `Cin = 1` no somador ao invés de adicionar 1 explicitamente:

```
A − B  =  A + ~B + 1   (com Cin = 1 no bit 0)
```

### Por que funciona?

Em aritmética de 8 bits (módulo 256):

```
~B = 255 − B   (inversão bit a bit)
~B + 1 = 256 − B ≡ −B (mod 256)
```

Portanto `A + ~B + 1 = A + (−B) = A − B`.

## Implementação: sub8bits.dig

O módulo `sub8bits.dig` usa dois componentes:

1. **inversor8bits.dig** — inverte todos os 8 bits de B usando uma porta NOT de 8 bits
2. **soma8bits.dig** — soma A + ~B com Cin=1

```
B[7:0] → [inversor8bits] → ~B[7:0] ─┐
                                      ▼
A[7:0] ──────────────────────────► [soma8bits] → Sub[7:0]
                          Cin=1 ──►              → neg (flag)
```

A flag **neg** indica que o resultado é negativo (A < B), detectado pelo sinal do bit mais significativo do resultado.

## Subtrator com Restauração de 1 bit (subRestore1bit.dig)

O **subtrator com restauração** é o componente central do algoritmo de divisão. Ele realiza a seguinte operação condicional:

- Tenta calcular `A − B`
- Se o resultado for negativo (Cout = 0), **restaura** o valor original de A (desfaz a subtração)
- Se o resultado for positivo ou zero (Cout = 1), mantém o resultado

```
Entradas: A, B, Cin, S (sinal de controle)
Saída:    R (resultado restaurado ou subtraído), Cout
```

O sinal S controla um multiplexor que seleciona entre o resultado da subtração e o valor original A. Internamente, usa o `soma1bit.dig` para a subtração condicional.

**Verificação pelo complemento de 2:**

```
  A:       0000 1010  (10)
  ~B:      1111 1001
  Cin=1:           +1
  ─────────────────
  ~B+1:    1111 1010  (−6 em complemento de 2)

  A + ~B + 1:
  0000 1010
+ 1111 1010
───────────
  0000 0100  = 4  ✓  (o carry-out de 9 bits é descartado)
```

:::info Resultado negativo
Se A < B, o carry-out do bit 7 será 0, e o resultado em complemento de 2 representará um número negativo. A flag `neg` é ativada para indicar essa condição.
:::
